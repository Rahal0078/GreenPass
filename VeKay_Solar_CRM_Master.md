## 1. System Overview

VeKay Solar CRM is a full-stack web application for managing solar installation complaints in Kerala, India. The brand used in emails is **GreenPass Technologies**.

### Three Portals

| Portal | URL | Who Uses It | Login |
|--------|-----|-------------|-------|
| Customer Portal | `/` | Customers submitting complaints | Not required |
| Admin Portal | `/admin` | Office staff, managers | Yes |
| Technician Portal | `/tech` | Field engineers | Yes |

### Complaint Lifecycle

```
Customer submits form
        │
        ▼
   status: open         ← just created, waiting for admin
        │
        ▼ (admin sets urgency + assigns tech)
   status: in_progress  ← technician assigned
        │
        ▼ (tech taps "Start Journey")
   status: going        ← on the way
        │
        ▼ (tech taps "I've Arrived")
   status: reached      ← on site
        │
        ▼ (tech taps "Complete Job" + writes notes)
   status: resolved     ← job done, email sent to customer
        │
        ▼ (admin closes)
   status: closed

   ← any stage can become → status: on_hold (with resume date)
```

### Tech Stack

| Layer | Technology |
|-------|-----------|
| Runtime | Node.js 24, TypeScript 5.9 |
| Package manager | pnpm workspaces (monorepo) |
| API server | Express 5 + express-session |
| Database | PostgreSQL (Neon cloud) + Drizzle ORM |
| Validation | Zod v4 + drizzle-zod |
| Frontend | React 18 + Vite, Tailwind CSS, Shadcn/ui |
| Routing | Wouter |
| Data fetching | TanStack React Query (generated by Orval) |
| Maps | react-leaflet + OpenStreetMap (free, no key) |
| Excel reports | exceljs |
| Email | nodemailer (Gmail SMTP) |
| Build | esbuild (CJS bundle) |
| Session store | connect-pg-simple (sessions in Postgres) |

## 2. SETUP GUIDE — Neon Database (Step by Step)

Neon is the free cloud PostgreSQL service this app uses. You need it to store all complaints, technicians, and admin accounts.

### Step 1 — Create a Neon Account

1. Open your browser and go to: **https://neon.tech**
2. Click **"Sign Up"** (top right)
3. You can sign up with Google, GitHub, or email
4. Complete email verification if prompted

### Step 2 — Create a Project

1. After logging in, click **"New Project"**
2. Fill in:
   - **Project name:** `vekay-solar-crm` (or any name)
   - **Region:** Choose **Asia Pacific (Singapore)** — closest to Kerala
   - **PostgreSQL version:** 16 (default is fine)
3. Click **"Create project"**
4. Neon creates your database in about 30 seconds

### Step 3 — Get the Connection String

1. You'll see a **"Connection Details"** panel
2. Look for the section labeled **"Connection string"**
3. It looks like this:
   ```
   postgresql://username:password@ep-xxxx-xxxx.ap-southeast-1.aws.neon.tech/neondb?sslmode=require
   ```
4. Click the **copy icon** next to it
5. **This is your `DATABASE_URL`** — keep it safe, treat it like a password

### Step 4 — Add to Replit Secrets

1. Open your Repl on Replit
2. Click the **lock icon (🔒 Secrets)** in the left sidebar
3. Click **"New Secret"**
4. Name: `DATABASE_URL`
5. Value: paste the connection string you copied
6. Click **"Add Secret"**
7. Repeat for `NEON_DATABASE_URL` (same value)

### Step 5 — Push the Database Schema

Once secrets are set, run this command in the Replit Shell:

```bash
pnpm --filter @workspace/db run push
```

This creates all the tables automatically:
- `complaints` — all customer complaints
- `technicians` — technician accounts
- `admins` — admin accounts
- `session` — login sessions

### Step 6 — Verify

```bash
# In Replit Shell, check the API is connecting:
curl localhost:80/api/health
# Should return: {"status":"ok"}
```

## 3. SETUP GUIDE — Gmail App Password (Step by Step)

The app sends automatic emails to customers (complaint registered, technician assigned, job completed). It also sends daily reports to `info.vekay@gmail.com`. This requires a Gmail App Password.

### What is an App Password?

It's a special 16-character password that lets the app send emails through Gmail without needing your real Gmail password. It's more secure than using the actual password.

### Requirements

- A Gmail account (the `info.vekay@gmail.com` account)
- 2-Step Verification must be turned ON

### Step 1 — Enable 2-Step Verification

1. Go to **https://myaccount.google.com**
2. Sign in as `info.vekay@gmail.com`
3. Click **"Security"** in the left menu
4. Under "How you sign in to Google", click **"2-Step Verification"**
5. Click **"Get started"** and follow the steps
6. Choose your verification method (phone number is easiest)
7. Complete the setup

### Step 2 — Generate an App Password

1. Go back to **https://myaccount.google.com/security**
2. Under "How you sign in to Google", look for **"App passwords"**
   - If you don't see it, make sure 2-Step Verification is fully enabled
3. Click **"App passwords"**
4. In the **"Select app"** dropdown, choose **"Mail"**
5. In the **"Select device"** dropdown, choose **"Other (Custom name)"**
6. Type: `VeKay Solar CRM`
7. Click **"Generate"**
8. Google shows a yellow box with a 16-character code like: `abcd efgh ijkl mnop`
9. **Copy this code immediately** — you cannot see it again

### Step 3 — Add to Replit Secrets

1. In Replit, click the **lock icon (🔒 Secrets)**
2. Click **"New Secret"**
3. Name: `GMAIL_APP_PASSWORD`
4. Value: paste the 16 characters **without spaces** (e.g. `abcdefghijklmnop`)
5. Click **"Add Secret"**

### Step 4 — Test Email

1. Log in to the Admin portal
2. Go to **Reports** tab
3. Click **"Send Email Report"**
4. Check `info.vekay@gmail.com` inbox — you should receive the daily report

### Emails the System Sends Automatically

| Trigger | Who Receives | Subject |
|---------|-------------|---------|
| New complaint submitted | Customer's email | "Complaint Registered Successfully" |
| Admin assigns technician | Customer's email | "Technician Assigned to Your Complaint" |
| Technician starts journey | Customer's email | "Your Technician is On the Way" |
| Technician arrives | Customer's email | "Technician Has Arrived at Your Location" |
| Job completed | Customer's email | "Your Complaint Has Been Resolved" |
| Admin clicks "Send Report" | `info.vekay@gmail.com` | "Daily Complaint Report — [date]" |

### If Emails Are Not Sending

1. Check that `GMAIL_APP_PASSWORD` secret is set correctly (no spaces)
2. Confirm the Gmail account has 2-Step Verification enabled
3. Try regenerating a new App Password
4. Check the API server logs in Replit for error messages
5. If `GMAIL_APP_PASSWORD` is missing, the app silently skips emails and still works normally

```sql
CREATE TABLE complaints (
  id                    SERIAL PRIMARY KEY,
  ticket_id             TEXT UNIQUE NOT NULL,
  customer_name         TEXT NOT NULL,
  customer_phone        TEXT NOT NULL,
  customer_email        TEXT NOT NULL,
  kseb_consumer_number  TEXT,
  place_name            TEXT NOT NULL,
  district              TEXT NOT NULL,
  pincode               TEXT NOT NULL,
  address               TEXT NOT NULL,
  lat                   REAL,
  lng                   REAL,
  location_source       TEXT,              -- 'gps' or 'map'
  complaint_type        TEXT NOT NULL,
  description           TEXT NOT NULL,
  media_urls            TEXT,              -- JSON array of image URLs
  status                TEXT NOT NULL DEFAULT 'open',
  urgency               TEXT NOT NULL DEFAULT 'low',
  urgency_color         TEXT,              -- hex color
  technician_id         INTEGER,           -- FK → technicians.id
  pending_technician_ids TEXT,             -- JSON array (broadcast jobs)
  admin_notes           TEXT,
  completion_notes      TEXT,
  scheduled_date        TEXT,              -- ISO date string
  completed_at          TIMESTAMPTZ,
  created_at            TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at            TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

### Auth Helpers — `artifacts/api-server/src/lib/auth.ts`

```typescript
import crypto from "crypto";

// Hash password with salt
export function hashPassword(password: string): string {
  return crypto.createHash("sha256")
    .update(password + "vekay_salt_2024")
    .digest("hex");
}

// Verify password against stored hash
export function verifyPassword(password: string, hash: string): boolean {
  return hashPassword(password) === hash;
}

// Generate unique ticket ID, e.g. "VK-20260601-A3B2C"
export function generateTicketId(): string {
  const date = new Date();
  const dateStr = date.toISOString().slice(0, 10).replace(/-/g, "");
  const rand = Math.random().toString(36).substring(2, 7).toUpperCase();
  return `VK-${dateStr}-${rand}`;
}
```

### Auth Routes — `artifacts/api-server/src/routes/auth.ts`

```typescript
// POST /api/auth/login — tries admin first, then technician
router.post("/auth/login", async (req, res) => {
  const { username, password } = req.body;

  // Try admin
  const [admin] = await db.select().from(adminsTable)
    .where(eq(adminsTable.username, username));
  if (admin && verifyPassword(password, admin.passwordHash)) {
    req.session.userId = admin.id;
    req.session.role = "admin";
    req.session.name = admin.name;
    req.session.username = admin.username;
    res.json({ id: admin.id, username: admin.username, name: admin.name, role: "admin" });
    return;
  }

  // Try technician
  const [tech] = await db.select().from(techniciansTable)
    .where(eq(techniciansTable.username, username));
  if (tech && verifyPassword(password, tech.passwordHash)) {
    if (!tech.isActive) {
      res.status(401).json({ error: "Account inactive. Contact admin." });
      return;
    }
    req.session.userId = tech.id;
    req.session.role = "technician";
    req.session.name = tech.name;
    req.session.username = tech.username;
    res.json({ id: tech.id, username: tech.username, name: tech.name, role: "technician" });
    return;
  }

  res.status(401).json({ error: "Invalid username or password" });
});

// POST /api/auth/logout
router.post("/auth/logout", (req, res) => {
  req.session.destroy(() => res.json({ message: "Logged out" }));
});

// GET /api/auth/me — check session
router.get("/auth/me", (req, res) => {
  if (!req.session.userId) {
    res.status(401).json({ error: "Not authenticated" });
    return;
  }
  res.json({
    id: req.session.userId,
    username: req.session.username,
    name: req.session.name,
    role: req.session.role,
  });
});
```

// Returns all jobs as map pins — includes direct assignments AND pending broadcasts
router.get("/technicians/:id/map", async (req, res) => {
  const techId = params.data.id;

  // Direct assignments
  const assigned = await db.select().from(complaintsTable)
    .where(eq(complaintsTable.technicianId, techId));

  // Pending (broadcast) — unassigned complaints that include this tech's ID
  const pending = await db.select().from(complaintsTable).where(
    and(
      sql`${complaintsTable.pendingTechnicianIds}::jsonb @> ${JSON.stringify([techId])}::jsonb`,
      sql`${complaintsTable.status} NOT IN ('resolved', 'closed')`,
      sql`${complaintsTable.technicianId} IS NULL`,
    )
  );

  // Return as map pins with isPending flag
  const pins = [
    ...assigned.map(c => toPin(c, false)),
    ...pending.map(c => toPin(c, true)),
  ];
  res.json(pins);
});

### Distance Calculation (Haversine Formula)

```typescript
function haversineKm(lat1: number, lng1: number, lat2: number, lng2: number): number {
  const R = 6371; // Earth radius in km
  const dLat = ((lat2 - lat1) * Math.PI) / 180;
  const dLng = ((lng2 - lng1) * Math.PI) / 180;
  const a = Math.sin(dLat / 2) ** 2
    + Math.cos((lat1 * Math.PI) / 180)
    * Math.cos((lat2 * Math.PI) / 180)
    * Math.sin(dLng / 2) ** 2;
  return R * 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1 - a));
}
```

## 20. Troubleshooting & Gotchas

### ❗ Always regenerate after changing the API spec

```bash
pnpm --filter @workspace/api-spec run codegen
```
Forgetting this causes TypeScript errors in the frontend hooks.

### ❗ Never run `pnpm dev` at the workspace root

Start services through Replit workflows only. Root has no `dev` script by design.

### ❗ Excel download uses fetch + blob, not React Query

```typescript
// Correct — direct fetch for file streaming
const res = await fetch("/api/reports/excel", { credentials: "include" });
const blob = await res.blob();
const url = URL.createObjectURL(blob);
const a = document.createElement("a");
a.href = url; a.download = "report.xlsx"; a.click();
```

### ❗ `useAuth()` import source

```typescript
// Correct
import { useAuth } from "@/lib/auth-context";

// Wrong — this is the API client hook, not auth
import { useAuth } from "@workspace/api-client-react";
```

### ❗ MapPin icon name conflict

```typescript
// Correct — alias the API type to avoid clash with lucide-react MapPin icon
import type { MapPin as ApiMapPin } from "@workspace/api-client-react";
import { MapPin } from "lucide-react";  // this is the icon, fine to use directly
type Pin = ApiMapPin;
```

### ❗ Orval v8 — always pass queryKey with enabled

```typescript
// Correct
useGetComplaint(id, {
  query: {
    enabled: !!id,
    queryKey: getGetComplaintQueryKey(id),  // required in Orval v8
  }
});
```

### ❗ Session cookie settings by environment

| Setting | Development | Production |
|---------|------------|------------|
| `secure` | `false` | `true` |
| `sameSite` | `"lax"` | `"none"` |

`sameSite: "none"` is required in production because the Replit preview proxy serves from a different origin than the API.

### ❗ `pnpm --filter @workspace/db run push` is for development only

Never run schema push directly on production. In production, use proper SQL migrations or the Neon SQL editor to apply schema changes manually.

### ❗ Passwords use SHA-256 + salt (not bcrypt)

```typescript
// The salt is hardcoded: "vekay_salt_2024"
// To generate a password hash manually:
crypto.createHash("sha256").update("yourpassword" + "vekay_salt_2024").digest("hex")
```

### ❗ To add a new admin account directly in DB

```sql
-- Get the hash first (Node.js):
-- require("crypto").createHash("sha256").update("newpassword" + "vekay_salt_2024").digest("hex")

INSERT INTO admins (username, password_hash, name, email)
VALUES ('newadmin', '<hash_from_above>', 'New Admin', 'admin@example.com');
```
